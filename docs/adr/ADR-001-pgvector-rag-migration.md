# ADR-001: pg_trgm에서 pgvector + RAG로 전환

## 상태
accepted

## 날짜
2026-05-27

## 맥락

v1 봇을 OZ 코딩스쿨 Discord에서 운영하면서 중복 질문 감지에 pg_trgm을 사용했다.
pg_trgm은 문자열 패턴 유사도 기반이라 **표현 방식이 다르지만 동일한 의미의 질문**을 중복으로 잡지 못하는 문제가 반복됐다.

예를 들어:
- "JWT 토큰 어떻게 저장해요?" vs "액세스 토큰 저장 방법이 뭐예요?"
- 의미는 같지만 pg_trgm 유사도 점수가 낮아 신규 질문으로 처리 → LLM 중복 호출

이로 인해 **불필요한 LLM 호출 증가** 및 **동일 답변 중복 저장** 문제가 발생했다.

## 결정

pg_trgm 중복 체크를 제거하고 **pgvector 기반 의미론적 유사도 검색 + RAG 파이프라인**으로 전환한다.

- 임베딩: OpenAI `text-embedding-3-small` (1536차원)
- 벡터 저장/검색: PostgreSQL + pgvector
- 유사도 임계값(SIMILARITY_THRESHOLD) 기반으로 중복 판단
- 신규 질문은 관련 문서를 검색한 컨텍스트와 함께 LLM에 전달 (RAG)

## 결과

**긍정적**
- 표현 방식이 달라도 의미 기반으로 중복 감지 가능
- RAG로 답변 품질 향상 (맥락 있는 답변)
- PostgreSQL 그대로 활용 → 인프라 추가 없음

**부정적/트레이드오프**
- 임베딩 생성 비용 발생 (OpenAI API)
- 유사도 임계값 튜닝 필요
- pg_trgm 대비 초기 세팅 복잡도 증가

# ADR-002: 답변 생성 모델을 Gemini로 분리

## 상태
accepted

## 날짜
2026-05-27

## 맥락

v1 봇을 클라우드(EC2)에 배포해서 운영했는데 **서버 비용 부담**이 발생했다.
또한 답변 생성에 OpenAI만 사용할 경우 API 호출 비용이 누적되는 구조였다.

v2 전환 시점에 비용 구조를 재검토했다.

## 결정

역할별로 모델을 분리한다.

| 역할 | 모델 | 이유 |
|------|------|------|
| 임베딩 | OpenAI text-embedding-3-small | 벡터 일관성, 업계 표준 |
| 답변 생성 | Gemini 2.5 Flash | 무료 티어 활용, 비용 절감 |
| 인프라 | 로컬 운영 | 클라우드 비용 제거 |

## 결과

**긍정적**
- 답변 생성 비용 사실상 0원 (Gemini 무료 티어)
- 임베딩은 OpenAI 유지로 벡터 품질 보장
- 모델 교체가 필요할 때 역할별로 독립적으로 변경 가능

**부정적/트레이드오프**
- 두 개의 API 키 관리 필요
- Gemini 무료 티어 한도 초과 시 대응 필요
- 로컬 운영이라 24/7 가동 불안정
